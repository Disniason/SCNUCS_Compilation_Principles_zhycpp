本仓库存放着本人完成华南师范大学计算机学院编译原理理论课程实验1~5时所编写的5个头文件，分别是：

"Zrust.h"：rust语言词法分析实验

"Zregexp.h"：词法分析器生成器实验

"Ztiny.h"：tiny扩充语言语法树生成实验

"Zlalr.h"：LALR(1)语法分析器生成器实验

"Ztiny_tuple.h"：tiny扩充语言中间代码生成实验

本仓库代码的优点包括如下几点：

1.高代码原创性和非AI率：

本仓库中所有代码的算法思路均由本人通过分析实验要求和课程提供的代码想出，没有借鉴网络中其它人的代码；所有的代码几乎由本人手动编写完成，代码非AI率达到98%。

2.相比课程提供的代码而言具有较高的现代性：

本仓库中的代码几乎完全摒弃C语言的编码风格，包括但不限于：(1)大量使用标准模板库；(2)完全摒弃裸指针，而是使用智能指针；(3)使用了std::filesystem::path打开文件，完全不用担心编码问题；(4)使用了std::numeric_limits、std::function、std::variant、匿名函数、强枚举等内容。

3.所有的头文件仅使用了标准库：

所有的定义和实现仅使用了标准库，这就意味着它能够在任何GUI框架项目中直接使用，比如QT和mfc。根本原因在于，如果在代码中使用了其它第三方库的内容，那就意味着它的通用性不强，比如使用QT实现的代码如果想要在mfc框架内使用则会相当麻烦，本代码则完全避免了这一问题。

4.不存在源文件和动/静态库：

本仓库仅由头文件组成，没有任何多余的源文件和动/静态库，那就意味着在你的项目中只需包含头文件就可以调用其内的函数，在链接编译时不需要加入相关的额外配置。综合3和4，本仓库的优点为简单易用，适合C++新手使用。为了避免多源文件包含该头文件之后导致的重定义问题，本头文件中所有的全局变量/常量、全局函数、类静态变量/常量、类静态函数均使用inline关键字约束。

5.没有命名冲突问题

本仓库中所有代码均在命名空间zhy以内，这就意味着所有的标识符不会与其它库和你自己定义的标识符有命名冲突。类似于万能头文件"bits/stdc++.h"，最后本人会将所有的头文件用一个新的头文件"zhycpp.h"包含。
